# Developing in Cascade

Developing in Cascade covers advanced topics of interest to web developers, including scripting, content migration, and web services development.

## [Formats](https://www.hannonhill.com/cascadecms/latest/developing-in-cascade/script-formats/index.html)

Formats are used to transform XML or API data into XHTML, XML, text, or other meaningful markup.

## [Velocity Tools](https://www.hannonhill.com/cascadecms/latest/developing-in-cascade/script-formats/velocity-tools.html)

Cascade CMS's Velocity Tools are a collection of functions that can be used to retrieve and manipulate content in conjunction with Apache Velocity Template Language.

## [Internal Cascade API](https://www.hannonhill.com/cascadecms/latest/developing-in-cascade/internal-cascade-api/index.html)

The internal Cascade API exposes a set of operations to developers for code run within Cascade CMS in Formats, Publish Triggers, Asset Factory Plugins, and other types of plugins.

## [Internal Cascade API Changelog](https://www.hannonhill.com/cascadecms/latest/developing-in-cascade/internal-cascade-api/internal-cascade-api-changelog.html)

## [REST API](https://www.hannonhill.com/cascadecms/latest/developing-in-cascade/rest-api/index.html)

Cascade CMS REST API is a lightweight API for interfacing with Cascade's Web Services.

## [SOAP Web Services](https://www.hannonhill.com/cascadecms/latest/developing-in-cascade/soap-web-services-api/index.html)

Web services provide Cascade CMS users with a powerful way to work with the system by providing a powerful back-end interface with which to interact.

## [SOAP Web Services Operations](https://www.hannonhill.com/cascadecms/latest/developing-in-cascade/soap-web-services-api/soap-web-services-operations.html)

## [SOAP Web Services Changelog](https://www.hannonhill.com/cascadecms/latest/developing-in-cascade/soap-web-services-api/soap-web-services-changelog.html)

Tracks changes to the SOAP Web Services API over time.

## [Code Sections](https://www.hannonhill.com/cascadecms/latest/developing-in-cascade/code-sections.html)

Cascade CMS currently supports the use of two types of code sections in the system: Passthrough code sections and protect code sections.

## [Publish Triggers](https://www.hannonhill.com/cascadecms/latest/developing-in-cascade/publish-triggers.html)

Publish Triggers are plug-ins that can be utilized during the publishing process. They allow developers to execute custom logic each time an asset (page, file, or other content item) is published.

## [System Pseudo-Tags](https://www.hannonhill.com/cascadecms/latest/developing-in-cascade/system-pseudo-tags.html)

System pseudo-tags are special system tags that instruct Cascade CMS to act on the content contained within them.
