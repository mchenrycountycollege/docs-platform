# Design in Cascade

Design in Cascade discusses Cascade CMS components involved in the process of building public-facing HTML pages and other publishable content.

## [Configurations](https://www.hannonhill.com/cascadecms/latest/design-in-cascade/configurations/index.html)

Configurations are collections of one or more outputs that are used to display page content.

## [Linking to Specific Outputs](https://www.hannonhill.com/cascadecms/latest/design-in-cascade/configurations/linking-to-specific-outputs.html)

## [Content Types](https://www.hannonhill.com/cascadecms/latest/design-in-cascade/content-types/index.html)

Content Types combine the look and feel of a page (Configurations) with the editable content fields of a page (Metadata Sets and Data Definitions) so that non-technical users can easily create and edit pages.

## [Editable Fields](https://www.hannonhill.com/cascadecms/latest/content-authoring/pages/editable-fields.html)

## [Data Definitions](https://www.hannonhill.com/cascadecms/latest/design-in-cascade/data-definitions/index.html)

A Data Definition is a collection of fields used in the creation and editing of page and block content.

## [Data Definition XML Schema Reference](https://www.hannonhill.com/cascadecms/latest/design-in-cascade/data-definitions/data-definition-xml-schema-reference.html)

## [Shared Fields](https://www.hannonhill.com/cascadecms/latest/design-in-cascade/data-definitions/shared-fields.html)

Shared Fields allow you to manage, update, and share fields and field groups across Data Definitions and across sites.

## [Smart Fields](https://www.hannonhill.com/cascadecms/latest/design-in-cascade/data-definitions/what-are-smart-fields.html)

With Smart Fields, you can selectively show fields and field groups in a Data Definition based on values selected in checkbox, drop-down, radio, and multi-select fields.

## [Metadata Sets](https://www.hannonhill.com/cascadecms/latest/design-in-cascade/metadata-sets/index.html)

Metadata is data within the CMS that describes an asset. A Metadata Set represents a curated collection of metadata fields used to describe content.

## [Metadata Fields in Cascade](https://www.hannonhill.com/cascadecms/latest/design-in-cascade/metadata-sets/metadata-fields-in-cascade.html)

## [Templates](https://www.hannonhill.com/cascadecms/latest/design-in-cascade/templates/index.html)

Templates are XHTML documents that provide the structure of page assets. Templates typically contain the HTML scaffolding and necessary scripts and CSS files to produce the desired "look and feel" of a page.

## [System Tags](https://www.hannonhill.com/cascadecms/latest/content-authoring/pages/system-tags.html)

Cascade CMS recognizes specific XML elements called system tags that are used for dynamic content insertion.
